5. Undo อย่างปลอดภัย

  1. หยุดงานที่กำลังเขียน dataset
  2. เปิดหน้า Import และตรวจ manifest ล่าสุด
  3. กด Undo แล้วอ่านรายการที่จะย้อนกลับ
  4. ยืนยัน และรอให้ระบบคืน image, label และ duplicate จาก quarantine
  5. รีเฟรชและตรวจจำนวนไฟล์ทั้งสาม split

Manifest

dataset\auto_improve\split_manifests

บันทึกต้นทาง ปลายทาง SHA และสถานะ transaction

Quarantine

dataset\auto_improve\split_quarantine\<manifest_id>

เก็บ exact duplicates เพื่อให้ตรวจสอบหรือ Undo ได้

กลไกป้องกันข้อมูล

เหตุการณ์พฤติกรรมระบบ
Dataset เปลี่ยนหลัง Previewหยุด Apply และตอบ 409 stale plan
ปลายทางมีไฟล์ชื่อเดียวกันหยุดก่อนเขียนทับและรายงาน conflict
SHA ต้นทางไม่ตรงหยุด transaction เพื่อรักษาไฟล์เดิม
ย้ายบางส่วนแล้วเกิดข้อผิดพลาดใช้ operation log เพื่อ rollback
ห้าม: ลบ manifest/quarantine หรือย้าย dataset ด้วย Windows Explorer ก่อนยืนยันว่าไม่ต้อง Undo

7. พร้อม Train หรือไม่

  • ทุกภาพที่ใช้มี YOLO label ถูกต้อง
  • กรอบชิดชิ้นงานและ class ถูกต้อง
  • Train และ Val มี label coverage เพียงพอ
  • ไม่มี hash หรือ capture group รั่วข้าม split
  • ทุก class มีตัวอย่างเพียงพอใน Train/Val
  • Test ไม่ถูกใช้ปรับค่าโมเดล
  • data.yaml มี path และ names ถูกต้อง
สถานะปัจจุบัน: ไม่พร้อม
มี label 1 ไฟล์จาก 1,249 ภาพ และ label นั้นอยู่ใน Test

แนะนำใช้จอเดสก์ท็อปเมื่อ Apply หรือ Undo เพื่ออ่านตาราง workpiece และคำเตือนได้ครบถ้วน

10. Checklist ผู้ปฏิบัติงาน

ก่อน Analyze

  • เปิดหน้า /import ถูกระบบ
  • หยุดงานที่เขียน dataset
  • ตรวจสถิติและ label coverage
  • กำหนด 80 / 10 / 10, gap 30, seed 42

ก่อน Apply

  • อ่าน warning ทุกบรรทัด
  • ตรวจ workpiece breakdown
  • ตรวจ duplicate และจำนวน moves
  • ยืนยันว่าไม่มี upload/annotate พร้อมกัน

หลัง Apply

  • บันทึก manifest ID
  • ตรวจ Train / Val / Test counts
  • สร้างและเปิด data.yaml
  • เก็บ manifest และ quarantine

ก่อน Train

  • Label coverage เพียงพอ
  • กรอบและ class ผ่าน QA
  • แต่ละ class มี Train/Val
  • Test ถูกกันออกจาก tuning
