3. เลือกค่าตั้งอย่างมีเหตุผล

ค่าแนวทางผลกระทบ
ModelNano สำหรับ smoke test; ขนาดใหญ่เมื่อข้อมูล/GPU พร้อมความเร็ว, VRAM, ความแม่นยำ
Epochs1-5 สำหรับทดสอบ pipeline; production ดู convergenceเวลาและ overfitting
Batchเพิ่มจนเหมาะกับ VRAM โดยไม่ OOMthroughput และ memory
Image Sizeเริ่ม 640; เพิ่มเมื่อวัตถุเล็กและทรัพยากรพอรายละเอียดและเวลา

ลำดับเริ่ม Run

1 QA Datasetcoverage และ leakage
2 Smoke Testยืนยัน path และ CUDA
3 Full Trainติดตาม log/metrics
4 Validateเลือก best checkpoint
backend local runner ตรวจไฟล์ label อีกชั้น แม้มีการเรียก API โดยไม่ผ่าน UI

Status และ Metrics

4. ติดตามผลระหว่างเทรน

สถานะ

  • idle: ยังไม่ทำงาน
  • training: กำลังรัน
  • completed: สำเร็จ
  • error: ตรวจ log ทันที

หลักฐาน

  • epoch และ progress
  • loss และ mAP
  • run name / config
  • best.pt / results.csv

Local และ Remote runner

ระบบตรวจ training service ก่อน หาก service offline จะใช้ local Ultralytics runner และแสดง runner: local หากออนไลน์จะส่งงานไป remote runner

อย่าปิด service หรือย้าย run directory ขณะกำลังเขียน weights

การอ่านผลเบื้องต้น

  • Loss ควรลดลงอย่างมีแนวโน้ม ไม่ใช้ค่าเดียวตัดสิน
  • mAP50 และ mAP50-95 ต้องพิจารณาร่วมกับตัวอย่างผิดพลาดจริง
  • ตรวจ per-class metric โดยเฉพาะ workpiece ที่มีตัวอย่างน้อย
